SUBNEG4 とは

SUBNEG4 は OISC (One Instruction Set Computer) の一種。命令はたった1つだけ:

A
引く値の
アドレス
B
引かれる値の
アドレス
C
結果の
格納先
D
負の場合の
ジャンプ先
mem[C] = mem[B] − mem[A]  ;  if result < 0 then goto D else PC += 4

この1命令だけで、加算・乗算・条件分岐・関数呼出・配列アクセスなど、すべてのコンピュータ操作を実現する。

アーキテクチャ

1 Parser — .ll テキストを構造化データに変換 parser.rb + ir_nodes.rb

LLVM IR テキストを行単位で正規表現パースし、IRModule → IRFunction → BasicBlock → IRInstruction のツリーを構築。

対応: define, 基本ブロックラベル, SSA代入 (%r = opcode ...), void命令 (ret, br, store), phi, switch (複数行), select, キャスト, tail call

IRModule IRFunction BasicBlock IRInstruction Operand (:var :const :label :global :raw :type)
▼
2 Lowering — IR命令を疑似命令に変換 lowering.rb

SSA形式のIR命令を、SUBNEG4に近い疑似命令列に変換。最も複雑なフェーズ (1,184行)。

主要タスク: メモリ割当 (alloca/グローバル), SSA変数→名前付き変数, phi節→分岐点コピー挿入, 比較→減算+符号判定, 間接アクセス→ポインタ追跡, 関数呼出→push/pop

PSub PAdd PCp PGoto PNeg PPush PPop PIndirectLoad PIndirectStore
▼
3 Expander — 疑似命令をSUBNEG4命令に展開 expander.rb

各疑似命令を1〜3個のSUBNEG4命令に機械的に展開。自己書き換えコードのパターンもここで生成。

例: PAdd(a,b,c) → [a, ZERO, t, NEXT] + [t, b, c, NEXT] (2命令で加算)

▼
4 Emitter — SUBNEG4 アセンブリテキスト出力 emitter.rb

データセクション (定数・変数・アドレス参照) + コードセクション (SUBNEG4命令) + スタック領域を出力。subneg4-vue デバッガと互換性のあるフォーマット。

疑似命令 → SUBNEG4 展開

すべての計算は「引き算して、結果が負ならジャンプ」の組み合わせで実現される。

疑似命令 意味 SUBNEG4 展開 命令数
PSub(a, b, c) c = b − a a, b, c, NEXT 1
PAdd(a, b, c) c = b + a a, ZERO, t, NEXT
t, b, c, NEXT		 2
PCp(dst, src) dst = src ZERO, src, dst, NEXT 1
PGoto(label) 無条件ジャンプ ZERO, C_n1, t, label 1
PNeg(val, label) val < 0 ならジャンプ ZERO, val, t, label 1
PPush(val) スタックに push SP→C書換 + 書込 + SP-- 3
PPop(dst) スタックから pop SP++ + SP→B書換 + 読出 3
PIndirectLoad(dst, addr) dst = mem[addr] addr→B書換 + 読出 2
PIndirectStore(addr, val) mem[addr] = val addr→C書換 + 書込 2

例: 加算 (a + b) の実現

SUBNEG4 には「引き算」しかないが、二重の否定で加算を実現する。

C Source
int c = a + b;
→
LLVM IR
%c = add i32 %a, %b
疑似命令
PAdd(a, b, c)
→
SUBNEG4 (2命令)
// Step 1: t = 0 - a = -a
a, ZERO, t, NEXT

// Step 2: c = b - (-a) = b + a
t, b, c, NEXT

メモリレイアウト

プログラムとデータが同じメモリ空間を共有する (フォン・ノイマン方式)。

0
ZERO:0 定数ゼロ (加算・コピーの基点)
1..N
定数プール C_1, C_n1, C_2, ... (定数値)
N+1..M
グローバル変数 counter:10, multiplier:3, ...
M+1..P
変数領域 関数パラメータ, ローカル変数, 一時変数, 配列
P+1..Q
コードセクション SUBNEG4 命令列 (4ワード/命令)
Q+1..末尾
スタック領域 256ワード (下方成長, SP → stack_top)

自己書き換えコード

SUBNEG4 にはレジスタもアドレッシングモードもない。間接メモリアクセス (ポインタ参照) は命令自身を書き換えることで実現する。

例: 間接読出 — dst = mem[ptr] (PIndirectLoad)
1
ptr の値を、次の命令の B オペランド に書き込む
ZERO, ptr, cb:read_b, NEXT // read_b のアドレスに ptr の値をコピー
2
書き換えられた命令を実行 → B が ptr の指す先を参照
ZERO, cb:read_b:???, dst, NEXT // dst = mem[ptr] - 0 = mem[ptr]

cb: はオペランド B のアドレスを示す「per-operand ラベル」。実行時に step 1 がこのアドレスに値を書き込み、step 2 の命令のソースアドレスが動的に変わる。

例: スタック Push (PPush)
1
SP の値を、書込命令の C オペランド に書き込む
ZERO, SP, cc:push_c, NEXT
2
val を mem[SP] に書き込む (C が SP の値に書き換え済み)
ZERO, val, cc:push_c:???, NEXT
3
SP をデクリメント (スタックは下方成長)
C_1, SP, SP, NEXT // SP = SP - 1

関数呼出の仕組み

ハードウェアスタックが存在しないため、ソフトウェアスタックと自己書き換えで再帰を実現。

Phi 節の変換

LLVM IR の SSA 形式では、合流点で phi 命令が異なるパスの値を選択する。S4C では phi を分岐点でのコピー命令に変換する。

LLVM IR (SSA形式)
loop:
  %i = phi i32 [0, %entry], [%i.next, %loop]
  %sum = phi i32 [0, %entry], [%sum.new, %loop]
  ...
→
S4C の変換結果
// entry → loop の分岐前:
CP(i, 0)
CP(sum, 0)
GOTO loop

// loop → loop のバックエッジ前:
CP(i, i_next)
CP(sum, sum_new)
GOTO loop

LLVM IR疑似命令備考
%r = add i32 %a, %b PAdd(a, b, r) 2命令 (否定して引く)
%r = sub i32 %a, %b PSub(a, b, r) 1命令 (直接)
%r = mul i32 %a, %b call __mul ループ加算サブルーチン
%r = sdiv i32 %a, %b call __sdiv ループ減算サブルーチン

%r = shl/ashr i32 %a, %b call __shl/__shr ループ 2倍/2分の1

組み込みサブルーチン

SUBNEG4 には乗算・ビット演算命令がないため、ループで実装するサブルーチンが自動生成される。

▶ __mul(a, b) → result — 乗算 (繰り返し加算) 
// result = 0
// while b > 0:
//   result += a
//   b -= 1
// 負数処理: b < 0 の場合は符号反転して最後にnegate
▶ __and(a, b) → result — ビットAND (32回ループ) 
// result = 0, bit = 1
// 32.times do:
//   a_bit = a - (a/2)*2  (最下位ビット)
//   b_bit = b - (b/2)*2
//   if a_bit == 1 && b_bit == 1:
//     result += bit
//   a >>= 1, b >>= 1, bit <<= 1
▶ __shl(a, n) → result — 左シフト (n回ループ) 
// result = a
// n.times do:
//   result = result + result  (= result * 2)

完全な変換例: add(3, 5)

C Source

int add(int a, int b) {
    return a + b;
}
int main() {
    return add(3, 5);
}

LLVM IR (.ll)

define i32 @add(i32 %0, i32 %1) {
  %3 = alloca i32       ; a
  %4 = alloca i32       ; b
  store i32 %0, ptr %3
  store i32 %1, ptr %4
  %5 = load i32, ptr %3
  %6 = load i32, ptr %4
  %7 = add i32 %5, %6
  ret i32 %7
}

SUBNEG4 Assembly (出力)

// === Data Section ===
ZERO:0
C_n1:-1
add_arg_0:0            // 引数 a
add_arg_1:0            // 引数 b
add___retval:0         // 戻り値
add_7:0                // a + b の結果
t0:0                   // 一時変数
...

// === Code Section ===
// add: %7 = add %5, %6
add:add_arg_0, ZERO, t0, NEXT   // t0 = -a
t0, add_arg_1, add_7, NEXT      // add_7 = b - (-a) = a + b

// ret i32 %7
ZERO, add_7, add___retval, NEXT // retval = a + b
ZERO, C_n1, t1, cd:add_ret_d:0 // return (D = caller が設定)
